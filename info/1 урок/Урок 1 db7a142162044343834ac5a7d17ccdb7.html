  types: [
    { value: 'feat', name: 'feat:      Добавление нового функционала' },

    { value: 'docs', name: 'docs:      Обновление документации' },

    { value: 'perf', name: 'perf:      Изменения направленные на улучшение производительности' },
    { value: 'refactor', name: 'refactor:  Правки кода без исправления ошибок или добавления новых функций' },

    Про папку core: 1. models: Модель - это класс, который определяет структуру и поведение объектов, представляющих определенный тип данных или сущность в приложении. 2. fabrics: Фабрика - это класс или функция, которая отвечает за создание экземпляров других классов или объектов. Ее основная задача заключается в абстрагировании процесса создания сложных объектов, чтобы упростить код и улучшить его читаемость. 3. adapters: Адаптер - это класс или функция, которая служит для преобразования данных из одного формата или структуры в другой. Его основная задача состоит в адаптации данных для использования в различных частях приложения. 4. services: Сервис - это класс или функция, которая предоставляет функциональность для выполнения определенных задач или операций в приложении. Он объединяет логику, связанную с определенной функциональностью, и предоставляет удобный интерфейс для взаимодействия с этой функциональностью. 5. methods: API-методы представляют функциональность, связанную с взаимодействием с сервером или сторонними API для получения или отправки данных. 6. configs - Любые конфигурации для работы приложения и/или работы с данными Архив с шаблоном:
